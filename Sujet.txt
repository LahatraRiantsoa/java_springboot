Développement Full-Stack : Application avec Backoffice
Contexte du projet
Vous devez développer une application complète avec backoffice sécurisé et interface frontend. Le projet mettra en œuvre vos compétences en développement full-stack avec Spring Boot et un framework frontend.

Partie 1: Backend Spring Boot
Modèles de données
Créez les entités suivantes:

User (utilisateur système)

id, username, email, password, roles (USER, ADMIN), enabled, createdAt
Product (produit)

id, name, description, price, category, stockQuantity, createdAt, lienImage
Categorie

id, name
Order (commande)

id, user, orderDate, totalAmount, status (PENDING, PROCESSING, SHIPPED, DELIVERED), produit, quantite
Sécurité Spring Security
Configurez Spring Security avec:

Authentification par JWT (JSON Web Token)
Deux rôles: USER et ADMIN
Les endpoints /api/admin/** accessibles uniquement aux ADMIN
Les endpoints /api/auth/** accessibles publiquement
Les endpoints /api/user/** accessibles aux utilisateurs authentifiés
'/' accessibles publiquement (afficher une image)
Implémentez:

Endpoint d'inscription (register)
Endpoint de connexion (login)
api
les roles sont enregistrer en dur
API REST Controller
Créez les controllers REST suivants:

AuthController (endpoints publics)

POST /api/auth/register - Inscription utilisateur
POST /api/auth/login - Connexion
AdminController (accessibles uniquement aux ADMIN)

CRUD complet pour les produits
GET /api/admin/users - Liste tous les utilisateurs
pouvoir modifier les informations d'un utilisateur
ProductController (accessibles publiquement en lecture)

GET /api/products - Liste tous les produits (paginé)
GET /api/products/{id} - Détail d'un produit
GET /api/products/search - Recherche par nom/catégorie
OrderController (accessibles aux utilisateurs authentifiés)

POST /api/orders - Créer une commande
GET /api/orders/my-orders - Voir ses commandes
